New modalities and flexible output dimensions
The model is based on Gemini and leverages its best-in-class multimodal understanding capabilities to create high-quality embeddings across:
Text: supports an expansive context of up to 8192 input tokens
Images: capable of processing up to 6 images per request, supporting PNG and JPEG formats
Videos: supports up to 120 seconds of video input in MP4 and MOV formats
Audio: natively ingests and embeds audio data without needing intermediate text transcriptions
Documents: directly embed PDFs up to 6 pages long
Beyond processing one modality at a time, this model natively understands interleaved input so you can pass multiple modalities of input (e.g., image + text) in a single request. This allows the model to capture the complex, nuanced relationships between different media types, unlocking more accurate understanding of complex, real-world data.

Like our previous embedding models, Gemini Embedding 2 incorporates Matryoshka Representation Learning (MRL), a technique that “nests” information by dynamically scaling down dimensions. This enables flexible output dimensions scaling down from the default 3072 so developers can balance performance and storage costs. We recommend using 3072, 1536, 768 dimensions for highest quality.

### Step 1: Basic Setup & Text Embedding
```python
import os
import google.generativeai as genai
import numpy as np

# Configure (use env var in production)
genai.configure(api_key=os.getenv("GEMINI_API_KEY") or "YOUR_API_KEY_HERE")

MODEL = "models/gemini-embedding-2-preview"

# Simple text embedding
result = genai.embed_content(
    model=MODEL,
    content="What makes a good RAG system?",
    task_type="RETRIEVAL_DOCUMENT"   # or RETRIEVAL_QUERY for queries
)

emb = np.array(result["embedding"])
print(f"Dimension: {emb.shape}")           # → (3072,)
print(f"Norm: {np.linalg.norm(emb):.4f}") # usually ~1 after internal normalization
```

**Tip**: For best cosine similarity, normalize if you truncate dimensions later.

### Step 2: Matryoshka → Flexible Dimensions
Reduce storage/latency while preserving most quality (MRL property).

```python
# Same text, but request smaller vector
result_768 = genai.embed_content(
    model=MODEL,
    content="What makes a good RAG system?",
    task_type="RETRIEVAL_DOCUMENT",
    output_dimensionality=768           # valid: 128–3072
)

emb_768 = np.array(result_768["embedding"])
print(f"Reduced dim: {emb_768.shape}")     # → (768,)
```

Common sweet spots: 768 (fast & cheap), 1536 (good balance), 3072 (max quality).

You can interleave up to 6 images + text in one call.

### Step 4: Video, Audio, PDF (File API – Async Upload)
Larger files require the **File API** (upload once, reference by URI).

```python
import time

def upload_and_embed(path: str):
    mime = ""
    if path.endswith(".mp4"): mime = "video/mp4"
    elif path.endswith(".mp3"): mime = "audio/mp3"
    elif path.endswith(".pdf"): mime = "application/pdf"
    else: raise ValueError("Unsupported file")

    file = genai.upload_file(path, mime_type=mime)
    
    # Poll until ACTIVE
    while file.state.name == "PROCESSING":
        print(f"Processing {path}...")
        time.sleep(4)
        file = genai.get_file(file.name)
    
    if file.state.name != "ACTIVE":
        raise RuntimeError(f"File failed: {file.state.name}")
    
    res = genai.embed_content(model=MODEL, content=file)
    emb = np.array(res["embedding"])
    print(f"{path} → {emb.shape}")
    return emb, file  # keep file ref if needed later

# Examples
vid_emb, vid_file   = upload_and_embed("./media/puppy.mp4")   # ≤120s
audio_emb, _        = upload_and_embed("./media/greeting.mp3") # ≤80s
pdf_emb, pdf_file   = upload_and_embed("./media/research.pdf") # ≤6 pages
```

**Important limits**:
- 1 video per request
- Audio + video together limited by total tokens
- PDFs auto-converted; max 6 pages

### Step 5: Cross-Modal Similarity – Quick Demo
```python
from sklearn.metrics.pairwise import cosine_similarity

# Assume we have embeddings from above
query_text = "cute puppy playing outside"
q_res = genai.embed_content(model=MODEL, content=query_text, task_type="RETRIEVAL_QUERY")
q_emb = np.array(q_res["embedding"])

# Normalize (recommended when using truncated dims)
q_emb /= np.linalg.norm(q_emb)

candidates = {
    "text":   genai.embed_content(model=MODEL, content="A dog runs happily", task_type="RETRIEVAL_DOCUMENT")["embedding"],
    "image":  res_img["embedding"],
    "video":  vid_emb,
    "pdf":    pdf_emb
}

for name, emb_raw in candidates.items():
    emb = np.array(emb_raw)
    emb /= np.linalg.norm(emb)
    sim = cosine_similarity([q_emb], [emb])[0][0]
    print(f"{name:8} similarity: {sim:.4f}")
```

You should see high similarity between semantically related items regardless of modality.

### Step 6: Full Multimodal RAG Pipeline (ChromaDB)
Real-world example: index mixed documents → retrieve → augment LLM.

```python
import chromadb
from chromadb.utils.embedding_functions import SentenceTransformerEmbeddingFunction  # fallback; we use custom

client = chromadb.PersistentClient(path="./multimodal_db")
collection = client.get_or_create_collection(
    name="gemini_multimodal_rag",
    metadata={"hnsw:space": "cosine"}
)

# Helper to add item
def index_item(id: str, content, metadata=None):
    if isinstance(content, (str, Image.Image)):
        res = genai.embed_content(model=MODEL, content=content)
    else:  # uploaded file
        res = genai.embed_content(model=MODEL, content=content)
    
    emb = res["embedding"]
    
    collection.add(
        ids=[id],
        embeddings=[emb],
        metadatas=[metadata or {}],
        documents=[str(content) if isinstance(content, str) else "[media]"]  # placeholder
    )

# Index examples
index_item("doc1", "RAG combines retrieval + generation for grounded answers")
index_item("img1", Image.open("./media/dog.jpg"), {"type": "image", "desc": "golden retriever"})
index_item("vid1", vid_file, {"type": "video", "duration": "45s"})
index_item("pdf1", pdf_file, {"type": "document"})

# Query → retrieve top-k
query = "animals having fun outdoors"
q_res = genai.embed_content(model=MODEL, content=query, task_type="RETRIEVAL_QUERY")
q_emb = q_res["embedding"]

results = collection.query(query_embeddings=[q_emb], n_results=3)

for i, (id, dist, meta) in enumerate(zip(results["ids"][0], results["distances"][0], results["metadatas"][0])):
    print(f"#{i+1}  ID: {id}  dist: {dist:.4f}  meta: {meta}")
```

Use the retrieved items (text + metadata + original files) to augment a Gemini generation call.

### Best Practices & Gotchas (2026 Preview)
- Always set `task_type` → RETRIEVAL_QUERY for user queries, RETRIEVAL_DOCUMENT for corpus items
- Normalize vectors when using output_dimensionality < 3072
- Handle File API cleanup: `genai.delete_file(file.name)`
- Batch embedding calls when possible (pass list to `content=`)
- Preview → monitor https://ai.google.dev/gemini-api/docs/changelog
- For scale → consider Vertex AI (paid, higher quotas, enterprise SLAs)
- Cost awareness: multimodal inputs are priced per modality (check pricing page)
