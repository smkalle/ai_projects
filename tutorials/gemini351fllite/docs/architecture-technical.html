state flow between stages
Data Flow & State Passing

All agents communicate through shared session state, not direct agent-to-agent passing. Each agent writes to its output_key. Downstream agents read via {key_name} interpolation in their prompts. 

sequenceDiagram
  participant U as User / CLI / UI
  participant S as Session State
  participant T as TriageAgent
  participant SY as SymptomAnalysis
  participant IM as ImagingAnalysis
  participant GN as DiagnosisGenerator
  participant CR as DiagnosisCritic
  participant RT as SpecialistRouter
  participant SP as Selected Specialist
  participant SM as CaseSummary

  U->>S: Seed patient_intake JSON
  Note over S: All keys pre-seeded as ""

  S->>T: {patient_intake}
  T->>S: triage_result

  par Parallel Execution
    S->>SY: {patient_intake}, {triage_result}
    SY->>S: symptom_analysis
  and
    S->>IM: {patient_intake}
    IM->>S: imaging_findings
  end

  loop max 3 iterations
    S->>GN: {patient_intake}, {triage_result}, {symptom_analysis}, {imaging_findings}, {diagnosis_critique}
    GN->>S: differential_dx
    S->>CR: {differential_dx}, {patient_intake}
    CR->>S: diagnosis_critique
    Note over CR: If confidence >= 0.80: escalate = True
  end

  S->>RT: {differential_dx}, {patient_intake}, {triage_result}
  RT->>SP: transfer_to_agent
  SP->>S: specialist_opinion

  S->>SM: All prior output_keys
  SM->>S: case_summary

adk patterns
ADK Pattern Reference
SequentialAgent
Outer pipeline. Runs 5 stages in strict order. Each stage completes before the next starts. State flows forward via output_key.
ParallelAgent
Stage 2. Two agents run concurrently. Both use include_contents='none' for stateless isolation — they read only from session state.
LoopAgent
Stage 3. Generator + Critic loop, max 3 iterations. Critic calls exit_diagnosis_loop which sets tool_context.actions.escalate = True to exit.
LLM Router
Stage 4. LlmAgent with sub_agents. The LLM decides which specialist to route to. ADK's transfer_to_agent handles routing automatically.
session state
Session State Model

All keys are pre-seeded as empty strings before the pipeline starts. This prevents template interpolation failures when agents reference keys that haven't been populated yet (e.g., {diagnosis_critique} on the first loop iteration).
