00 · Origin

Why agy breaks on mobile Linux

Two separate failure modes on two separate runtimes — both root-caused to the same thing: a glibc-linked Go binary that assumes a full Linux userland.

Failure Log — UserLAnd / proot
MmapAligned() failed — unable to allocate with tag
hint=0x4ed1c0000000, size=1073741824, alignment=1073741824
→ tcmalloc requires 1 GB aligned VA reservation; Android restricts proot VA space
Failure Log — Termux (native)
CANNOT LINK EXECUTABLE "/data/.../agy"
library "libresolv.so.2" not found: needed by main executable
→ agy binary is glibc-linked; Termux uses bionic. No glibc present.
💡 The insight: agy is a Go binary with glibc + tcmalloc dependencies. But its capabilities — multi-step reasoning, subagent dispatch, tool calling, JSON hooks — are all available through the Gemini Managed Agents API and the antigravity-sdk-python. Python has no such linking constraints and runs cleanly on any ARM64 Android environment.
01 · Product

Agylity — What it is

A lightweight proxy layer that replicates the core agy UX surface using managed APIs, Python SDK, and a minimal web UI — deployable on any Android proot Debian environment or remote server.

⚡
CLI Shim
A Python script aliased to agy that accepts the same flags as Antigravity CLI: --print, --continue, --model, --add-dir. Zero Go, zero glibc.
🌐
Web TUI
A React-based terminal-aesthetic web app served locally. Conversation history, subagent status, tool call log, artifact viewer. Accessible from the phone browser.
🤖
Managed Agent Bridge
Uses the Gemini Managed Agents API — one API call spins a full isolated Linux sandbox with reasoning loop, code execution, file system, and tool dispatch managed by Google.
🪝
JSON Hook Engine
Implements the same JSON hook format as agy: intercept before tool call, after model call, on loop stop. Drop-in compatible with existing .hooks/ configs.
📦
Skills / GEMINI.md
Reads GEMINI.md and .agents/agents.md from working directory. Injects them as system context per session — same convention as native agy.
🔀
Model Switching
Supports /model mid-session. Routes to Gemini 3.5 Flash by default; configurable to Gemini 3.1 Pro, Claude 3.5 Sonnet, or other supported backends via API key.

04 · Hook Engine

JSON Hook Engine

Agylity implements the same JSON hook format as native agy — intercept lifecycle events, modify behavior, short-circuit execution.

Hook Lifecycle Events

// .hooks/agylity.json — drop-in compatible with agy hook format { "hooks": [ { "event": "before_tool_call", // before any tool is invoked "match": { "tool": "shell_exec" }, // optional filter "action": "confirm" // confirm | block | log | mutate }, { "event": "after_model_call", // after each LLM turn "action": "log", "log_to": "~/.agylity/turns.jsonl" }, { "event": "on_loop_stop", // agent decides it's done "action": "notify", // POST to webhook URL "webhook": "http://localhost:9999/done" }, { "event": "after_file_edit", // after any file write "action": "log" } ] }

Hook Actions

ActionBehaviourApplies to
confirmPause execution, prompt user Y/N before proceedingbefore_tool_call
blockCancel the tool call entirely, return synthetic error to agentbefore_tool_call
logAppend event JSON to specified log file (or default)all events
mutateApply JSON Patch to the tool call args before executionbefore_tool_call
notifyPOST event payload to a webhook URLall events
05 · Web TUI

Web TUI Spec

A terminal-aesthetic React app served at localhost:7474. Designed for use from the Android phone browser when working in proot. IBM Plex Mono. Dark. Dense.

Layout Panels

💬
Conversation Panel
Full message thread with user / assistant turns. Markdown rendering. Code blocks with syntax highlight. Streaming via SSE.
🔧
Tool Call Log
Right sidebar. Shows every tool invoked by the agent: name, args (collapsed), result status, duration. Hook intercepts shown inline.
📋
Artifact Viewer
Files written by the agent appear as tabbed artifacts — HTML renders in iframe, code files syntax-highlighted, CSVs as table. Download button per artifact.
🤖
Subagent Tracker
Status bar cards for each background subagent: progress indicator, model, current tool, elapsed time. Click to expand full subagent thread.

Slash Commands (Web TUI)

# Available in the prompt input bar /model gemini-3.5-flash # switch model mid-session /agent "refactor handlers" # spawn background subagent /fork # fork conversation at current turn /rewind 3 # rewind 3 turns /clear # clear current context /export md # export as markdown /fast # skip planning step, direct execution /permissions # show/edit tool permission config

07 · SDK Integration

SDK Integration Pattern

How Agylity talks to the Gemini Managed Agents API via antigravity-sdk-python.

# agylity/agent_bridge.py — core integration from antigravity import Agent, AgentConfig from antigravity.hooks import HookRunner import asyncio, json, pathlib async def create_session(config: dict) -> str: # Read project context files ctx = "" for p in ["GEMINI.md", ".agents/agents.md"]: if pathlib.Path(p).exists(): ctx += pathlib.Path(p).read_text() agent = Agent(AgentConfig( model=config.get("model", "gemini-3.5-flash"), system_instruction=ctx, managed=True, # ← Managed Agents API tools=["code_execution", "file_system", "web_search"], auto_approve=config.get("auto_approve", False), )) hook_runner = HookRunner.from_json(config.get("hooks_path")) return await agent.create_session(hooks=hook_runner) async def stream_prompt(session_id: str, prompt: str): async for event in session.stream(prompt): yield event # text_delta | tool_call | tool_result | done
🔑 Auth: Set GEMINI_API_KEY env var in proot Debian. The Managed Agents API runs on the Gemini API billing tier — free during preview with generous rate limits, same model as the agy binary would use. No GCP project required for personal use.
